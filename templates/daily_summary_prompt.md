## 输出形态（必须遵守）

- **总结感优先**：读者 60 秒内能抓住「今天主线条 + 关键事实 + 一句态度」，不要写成流水账或事件复述。
- **结构化**：全文用二级、三级标题分层；**优先用短列表（`-`）**，少用长段落；每个列表项尽量一行说完（建议 ≤45 字/条）。
- **简洁**：全文建议 **800～1600 字**（中文）；与结论无关的细节不写；**禁止**逐条复述 `events` JSON、禁止堆叠 URL/标题长表。
- **可追踪**：关键判断尽量带一句依据（例如「约 N 条浏览器事件」「滴答合计 M 分钟」），但不要罗列明细。

## 口吻与价值观

开头必须使用类似这样的口吻，但不要照抄：

「{opening_hint}」

- 少文学化，少比喻；温暖克制，不鸡血，不 PUA，不做效率审判。
- 不使用 emoji。
- 可温和指出信息摄入偏高、专注纯度、恢复不足。

## 章节结构（按顺序，标题照抄）

### 1. 今日摘要

用 **3～5 条** bullet，概括：今天的主线任务、一个亮点、一个风险或消耗点（若有）、一句收束。

### 2. 时间与节奏

按 **3～4 个时段**（如 09–12 / 12–18 / 18–24，可按数据密度合并）各写 **2～4 条** bullet。  
每条格式建议：`做了什么 · 主要平台/工具 · 关联维度（若有）`。不写表格、不贴长链接。

### 3. 成长维度快照

按用户配置的维度，**每个维度最多 3 条** bullet；无显著活动则写「今日痕迹较少」一句带过。

### 4. 输入、专注与恢复

- **信息摄入**：低/中/高 + 一句理由（可引用预统计中的事件量、时段数等）。
- **聚焦**：一句话；可引用 `focus_vs_information_flow` 中的数字作**方向性**判断，避免道德化。
- **健康与恢复**：睡眠/运动/步数有一句即可；无数据则写「当日未导入健康数据」。

### 5. AI 工具与工程

用 **短列表** 写 Cursor / VS Code / ChatGPT / 豆包 / DeepSeek 等：**有无使用、主要用途、一句效果判断**；无则写「未检测到」。

### 6. 手机端信息流（若有）

B 站、知乎、小红书等：**主题倾向一句 + 若明显偏多再补一句**；未导入则写「未导入」。

### 7. 今天不是白过

**3 条以内** bullet：输入 / 思考 / 输出 / 积累，各用极短句概括。

### 8. 一句摄影机

**一句话**，像镜头定帧，不展开。

### 9. 数据置信度

**2～4 条** bullet：哪些来源可靠、哪些明显缺口；不写技术细节。
