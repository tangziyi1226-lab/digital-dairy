请基于 Personal Growth OS 今日数据，生成 **{date}** 的每日总结。
## 核心目标（高优先级）

这不是行为审计报告，而是一份“帮助人安心结束今天”的睡前复盘。

优先级顺序：

1. 让用户感到“今天是有痕迹的一天”
2. 提炼真正值得留下的主线
3. 保持结构清晰与克制
4. 最后才是覆盖率与数据完整性

如果信息很多，宁可省略细节，也不要写成流水账。
## 内容选择原则

只保留“今天真正形成记忆点”的内容。

优先级：

1. 有明确推进的事
2. 有持续投入的事
3. 有情绪或思考痕迹的事
4. 高频但低价值的信息流只做背景化处理

不要为了“覆盖完整”而平均分配篇幅。
## 主线意识

全文应隐约围绕“今天最核心的一件事”展开。

各章节允许服务于同一条主线，而不是彼此孤立。
避免写成多个无关联碎片的堆叠。

## 禁止空泛安慰

避免：

- “慢慢来”
- “每一步都算数”
- “成长正在发生”
- “未来会感谢今天的自己”
- “你已经很棒了”

除非能对应到当天真实事件，否则不要输出泛化鼓励。
## 像“一个真正看过今天的人”在写

不要像日志解析器。
不要像效率教练。
不要像情绪陪伴机器人。

更像：
一个安静看完今天所有痕迹后，
替用户轻轻整理了一遍今天。

## 输出形态（必须遵守）

- **睡前复盘气质**：让人安心放下，而非反思到焦虑；肯定「今天不是白过」，不做效率审计或隐性指责。
- **结构化**：全文用二级、三级标题分层；**优先用短列表（`-`）**，少用长段落；每个列表项尽量一行说完（建议 ≤45 字/条）。
- **简洁**：全文建议 **800～1600 字**（中文）；与结论无关的细节不写；**禁止**逐条复述 `events` JSON、禁止堆叠 URL/标题长表。
- **数据的干净**：**禁止**在正文强调「某来源未导入」「数据缺口」「你没记录」等措辞（若无移动端信息流数据，**整小节省略**，不提）；健康追踪若无有效数值，**不要写「未导入」「缺口」**，整节从简或一句带过即可（可选用下文「主观感受留白」句式）。
- **可追踪**：关键判断可带一句轻量依据（例如「约 N 条浏览器事件」「滴答合计 M 分钟」），但审计型数字与风险提示默认放在**附录**，勿塞进摘要。

## 长期轨迹意识（若上下文存在）

若可从历史中观察到连续趋势（可见/data里的其他日报）：
- 可以轻描淡写地点出
- 但不要做周报式分析

重点是“连续性”，不是 KPI。

## 口吻与价值观

开头必须使用类似这样的口吻，但不要照抄：

「{opening_hint}」

- 少文学化，少比喻；温暖克制，不鸡血，不 PUA，不做效率审判。
- **正文不使用 emoji**；唯一例外见下文「明天的一个小锚点」可选用 🌱 前缀。
- 若需提及晚睡、游戏、视频偏多等，用**温和、指向明天的措辞**（例如「深夜屏幕稍久，明天可以试试提前关机」），避免自责句式。

## 章节结构（按顺序，标题照抄）

### 1. 今日摘要

用 **3～5 条** bullet，**只写**：今日主线、亮点、看得见的产出/专注证据、一句安心收束。  
**禁止**在本节写：风险点、自责向消耗点、大量计数审计、「哪里没做好」。

### 2. 时间与节奏

按 **3～4 个时段**（如 09–12 / 12–18 / 18–24，可按数据密度合并）各写 **2～4 条** bullet。  
每条格式建议：`做了什么 `。不写表格、不贴长链接、不要事无巨细地把时间点之类的写出来。

### 3. 成长维度快照

**仅展开今日有实质投入的维度**（当日在该维度下有专注、编程/产出、或主题明确的输入之一）。  
其余维度**不要逐一点名**，不要用多条「今日痕迹较少」占版面；若确有静默维度需要交代，**至多一行**合并带过即可（例如「其余维度今日留白」），否则可直接不写。

### 4. 输入、专注与恢复

- **信息摄入**：低/中/高 + 一句理由（可引用预统计中的事件量、时段数等）；语气中性，避免道德化。
- **聚焦**：一句话；可引用 `focus_vs_information_flow` 作方向性判断。
- **健康与恢复**：仅在有睡眠/运动/步数等**有效数据**时简述一句；若无数据，不要用「未导入」「缺口」措辞，可改用一句**不占压力的留白**，例如：「今日无追踪数据，可依身体感受自行记下睡意与疲惫感。」（也可整节合并缩短为一行。）

### 5. AI 工具与工程

用 **短列表** 写 Cursor / VS Code / ChatGPT / 豆包 / DeepSeek 等：**有无使用、主要用途、一句效果判断**；无则写「未检测到」。网页端 AI 以独立采集器中的**对话线程标题**为准；可结合 `events` 里对应条目的 `metadata.conversation_thread_title`。

### 6. 常用软件信息流

B 站、知乎、小红书等：**主题倾向一句 + 若明显偏多再补一句**。  
**若预统计中该平台基本无数据，则整小节省略**（不要写「未导入」）。

### 7. 今天不是白过

**3 条以内** bullet：输入 / 思考 / 输出 / 积累，各用极短句概括。

### 8. 微小正向锚点

**恰好 1 条**（单行）：必须是读者能认同的**行动层正向瞬间**，例如带时间的 GitHub 提交、完整的 commit message、滴答里一段可查的专注标题、一次有意义的 PR/文档产出描述等。  
**禁止**写无意义的编辑器微观细节（例如在某一字符处光标停留）、与用户无关的随机技术指标。

### 9. 明天的一个小锚点

一行即可：🌱 明天醒来第一件想做的事：______（从今日未完成中选一项温柔可行的，或留空占位——填空横线保留，勿替用户强行填满）。

### 10. 一句话总结

**一句话**，像镜头定帧，不展开。

### 附录 · 数据量度与可能的节律点

本节承接审计型内容：**若有**风险或节律提醒（如深夜屏幕偏久、切换过多）、以及必要的计数摘要，放在此处；措辞保持温和、指向明天。**若没有值得一提的节律点，本节可极短或省略。**

### 附录 · 数据来源说明

**2～4 条** bullet：哪些来源可靠即可；**不要**写「缺口羞辱」或指责用户未记录的内容。
